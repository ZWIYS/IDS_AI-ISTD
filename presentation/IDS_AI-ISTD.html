IDS_AI-ISTD

Система обнаружения вторжений на основе анализа сетевого трафика

Плювков П., Куликов А., Оглодин А., Пастухов В., Тойкина Д.

Введение

IDS (Intrusion Detection System) — это система обнаружения вторжений, предназначенная для анализа сетевого трафика и выявления подозрительной активности.

Основные задачи системы:

  • обнаружение аномалий в сетевом трафике
  • выявление потенциальных атак
  • анализ поведения пользователей
  • визуализация результатов анализа

Цель проекта

Целью проекта является разработка прототипа системы обнаружения вторжений, которая:

  • анализирует сетевой трафик
  • выявляет отклонения от нормального поведения
  • применяет методы анализа данных и машинного обучения
  • предоставляет удобную визуализацию результатов

Архитектура системы

Система состоит из нескольких ключевых компонентов:

  1. Сбор данных
  2. Обработка данных (ETL)
  3. Feature Engineering
  4. Аналитика и машинное обучение
  5. Визуализация результатов

Общий pipeline системы:

Network Traffic -> Data Collection -> ETL Processing -> Feature Engineering ->
-> Machine Learning -> Anomaly Detection -> Dashboard

Сбор данных

Источники данных, используемые в системе:

  • NetFlow — информация о сетевых потоках
  • PCAP — захваченные сетевые пакеты
  • системные журналы

Данные содержат информацию о:

  • IP адресе источника
  • IP адресе назначения
  • количестве пакетов
  • объеме переданных данных
  • продолжительности соединения

ETL обработка данных

На этапе ETL выполняются следующие задачи:

  • очистка данных
  • удаление некорректных или пустых значений
  • нормализация данных
  • подготовка данных для анализа

Этот этап позволяет подготовить данные для дальнейшего анализа и построения моделей.

Feature Engineering

Из сетевого трафика извлекаются ключевые признаки, необходимые для анализа:

  • количество соединений
  • общий объем переданных данных
  • длительность соединений
  • используемые порты
  • частота обращений к ресурсам Эти признаки используются в дальнейшем для обнаружения аномалий.

Машинное обучение

Для обнаружения аномалий могут применяться методы машинного обучения:

  • кластеризация (K-means)
  • Isolation Forest
  • анализ главных компонент (PCA)

Основная задача моделей — определить нормальное поведение сети и выявить отклонения от него.

Визуализация

Для представления результатов используется интерактивный dashboard.

Dashboard позволяет:

  • отображать статистику сетевого трафика

  • показывать обнаруженные аномалии

  • отслеживать подозрительные IP-адреса

  • анализировать динамику трафика во времени

Примеры обнаруживаемых угроз

Система может обнаруживать следующие типы подозрительной активности:

  • сканирование портов
  • DDoS атаки
  • резкие всплески сетевого трафика
  • подозрительные соединения

Такие события отмечаются системой как потенциальные аномалии.

Заключение

В рамках проекта была разработана концепция системы обнаружения вторжений.

Система:

  • анализирует сетевой трафик
  • выявляет аномальное поведение
  • использует методы анализа данных
  • предоставляет визуализацию результатов

В дальнейшем систему можно расширить:

  • интеграцией с системами мониторинга безопасности
  • использованием более сложных моделей машинного обучения
